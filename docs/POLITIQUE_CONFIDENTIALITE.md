## 3. Finalités du Traitement

Les données personnelles sont collectées et traitées pour les finalités suivantes :

### 3.1 Finalité principale
- **Prise en charge médicale** de l'enfant dans le cadre d'une intervention chirurgicale (circoncision)
- **Constitution du dossier médical** conformément aux obligations légales
- **Suivi pré et post-opératoire** du patient

### 3.2 Finalités secondaires
- **Signature électronique** des documents légaux (consentement éclairé, autorisation parentale, devis)
- **Gestion des rendez-vous** (pré-consultation, acte chirurgical)
- **Envoi de prescriptions médicales** par voie électronique
- **Communication** avec les parents concernant le suivi médical

### 3.3 Finalités techniques
- **Sécurité** de la plateforme et des données
- **Traçabilité** des accès aux données de santé (obligation HDS)
- **Amélioration** continue du service

## 4. Bases Légales du Traitement

Conformément au RGPD (article 6) et à la réglementation française sur les données de santé, les traitements sont fondés sur :

| Base légale | Traitements concernés |
|-------------|----------------------|
| **Consentement explicite** (art. 6.1.a + art. 9.2.a) | Signature électronique, communication par email/SMS |
| **Exécution d'un contrat** (art. 6.1.b) | Prise de rendez-vous, facturation |
| **Obligation légale** (art. 6.1.c) | Dossier médical, conservation 20 ans, traçabilité HDS |
| **Intérêt vital** (art. 6.1.d) | Urgences médicales |
| **Intérêt légitime** (art. 6.1.f) | Sécurité informatique, prévention fraude |

**Pour les données de santé** (article 9 RGPD), le traitement est autorisé sur la base de :
- Article 9.2.a : Consentement explicite de la personne concernée
- Article 9.2.h : Finalités de médecine préventive, diagnostic médical, prise en charge sanitaire

## 6. Durée de Conservation des Données

| Type de données | Durée de conservation | Base légale |
|-----------------|----------------------|-------------|
| **Dossier médical** | 20 ans à compter de la dernière consultation (ou jusqu'aux 28 ans du patient si mineur) | Article R. 1112-7 du Code de la Santé Publique |
| **Documents signés** (consentement, autorisation) | 20 ans | Preuve juridique |
| **Prescriptions médicales** | 20 ans | Dossier médical |
| **Données de connexion** | 1 an | LCEN + obligation HDS |
| **Logs d'audit** | 1 an minimum | Obligation HDS |
| **Compte utilisateur inactif** | 3 ans après dernière connexion, puis archivage | CNIL recommandation |

À l'issue de ces délais, les données sont :
- Soit **supprimées définitivement**
- Soit **anonymisées** pour usage statistique

### 7.2 Droit de rectification (article 16)

**Comment exercer ce droit :**
- Via la plateforme : Menu "Mon compte" → "Modifier mes informations"

### 7.3 Droit à l'effacement (article 17)
Vous pouvez demander la suppression de vos données personnelles.

**Limitations :**
- Les données du **dossier médical** ne peuvent pas être supprimées avant le délai légal de 20 ans

**Comment exercer ce droit :**

- Délai de réponse : 30 jours maximum

### 7.4 Droit à la portabilité (article 20)
Vous pouvez récupérer vos données dans un format structuré (JSON) pour les transférer à un autre praticien.

**Comment exercer ce droit :**
- Via la plateforme : Menu "Mon compte" → "Exporter mes données"
- Par email : [EMAIL À REMPLIR]

### 7.5 Droit d'opposition (article 21)
Vous pouvez vous opposer au traitement de vos données pour des motifs légitimes.

**Limitations :**
- Ne s'applique pas aux traitements fondés sur une obligation légale (dossier médical)

### 7.6 Droit de limitation (article 18)
Vous pouvez demander le gel temporaire du traitement de vos données.

Vous pouvez retirer votre consentement à tout moment pour les traitements fondés sur celui-ci (ex : communication par email).

**Comment exercer ce droit :**

## 8. Sécurité des Données

### 8.1 Mesures techniques

| Mesure | Description |
|--------|-------------|
| **Chiffrement au repos** | AES-256 pour tous les documents et données sensibles |
| **Chiffrement en transit** | TLS 1.3 (HTTPS obligatoire) |
| **Authentification forte** | MFA (authentification multi-facteurs) pour les praticiens |
| **Mots de passe** | Minimum 12 caractères, complexité obligatoire, hashage bcrypt |
| **Contrôle d'accès** | Séparation des rôles (patient/praticien), principe du moindre privilège |

### 8.2 Mesures organisationnelles

| Mesure | Description |
|--------|-------------|
| **Traçabilité** | Logs de tous les accès aux données de santé |
| **Sauvegardes** | Quotidiennes, chiffrées, géo-redondantes |
| **Tests de sécurité** | Audits réguliers, veille sécuritaire |
| **Formation** | Sensibilisation du personnel aux bonnes pratiques |

### 8.3 Certification HDS
L'hébergement est assuré par Microsoft Azure, certifié **Hébergeur de Données de Santé (HDS)** par un organisme accrédité, conformément aux articles L. 1111-8 et R. 1111-8-8 et suivants du Code de la Santé Publique.
